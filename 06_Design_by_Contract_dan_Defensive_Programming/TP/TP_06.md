## Soal
Diberikan dua kode yang sama-sama melakukan operasi pembagian. Pertama menggunakan asersi, kedua menggunakan eksepsi.
```
const assert = require('assert');

function divide(a, b) {
  assert(typeof a === 'number' && typeof b === 'number', 'Nilai harus bilangan bulat');

  assert(b !== 0, 'Tidak bisa pembagian dengan nol');

  return a / b;
}
function divide(a, b) {
  if (typeof a !== "number" || typeof b !== "number") {
    throw new TypeError("Nilai harus bilangan bulat");
  }

  if (b === 0) {
    throw new Error("Tidak bisa pembagian dengan nol");
  }

  return a / b;
}

try {
  const result = divide(10, 2);
  console.log("Hasilnya adalah:", result);
} catch (error) {
  console.error("Error:", error);
}
```
Menurutmu, kapankah kita saatnya menggunakan asersi atau eksepsi untuk fungsi seperti ini di atas? Apakah kita harus sepenuhnya asersi, atau sepenuhnya eksepsi? Lakukan riset dan berikan jawabannya dalam bentuk esai minimal 300 kata.


## Esai
Ketika kita memutuskan untuk menggunakan asersi atau eksepsi sebenarnya tidak bisa dipisahkan dari konteks penggunaannya, misal dari mana data berasal dan bagaimana sistem menangani kesalahan secara keseluruhan. Kedua mekanisme ini memiliki tujuan yang berbeda asersi digunakan untuk mendeteksi bug internal dalam kode, sedangkan eksepsi digunakan untuk menangani kondisi error yang terjadi ketika program yang kita bikin di jalankan 
Asersi berkaitan dengan konsep Design by Contract (DbC) yang diperkenalkan oleh Bertrand Meyer. Dalam pendekatan ini, setiap fungsi memiliki precondition atau syarat awal yang harus dipenuhi oleh yang manggil fungsi tersebut. Misalnya, pada fungsi pembagian, nilai input harus berupa angka dan pembagi tidak boleh nol. Jika kondisi ini dilanggar, maka hal tersebut menunjukkan adanya kesalahan logika pada kode pemanggil. Oleh karena itu, asersi digunakan untuk memastikan asumsi-asumsi internal tetap benar dan menerapkan prinsip fail-fast yang dapat menghentikan program lebih awal agar bug dapat segera terdeteksi.
Namun, penggunaan asersi tidak selalu cocok untuk semua situasi contohnya Dalam lingkungan produksi, asersi sering kali tidak digunakan untuk mekanisme utama penanganan error karena dapat menyebabkan program berhenti secara tiba-tiba. Selain itu, dalam beberapa kasus, asersi bahkan dapat dinonaktifkan untuk menjaga performa.
Di sisi lain, eksepsi merupakan bagian dari defensive programming. Eksepsi digunakan untuk menangani kesalahan yang memang mungkin terjadi, seperti input yang tidak valid dari pengguna atau kondisi tak terduga lainnya. Berbeda dengan asersi, eksepsi dirancang untuk dapat ditangani menggunakan mekanisme try-catch, sehingga program tetap dapat berjalan atau memberikan respons yang sesuai tanpa harus berhenti secara tiba-tiba.
Pada praktiknya pendekatan yang paling baik adalah menggabungkan keduanya. Pada bagian luar sistem, yang berinteraksi langsung dengan user atau sistem lain, eksepsi sebaiknya digunakan untuk melakukan validasi dan menangani error. Sementara itu, di bagian internal sistem, asersi dapat digunakan untuk menjaga konsistensi logika dan memastikan bahwa asumsi antar-komponen tetap terpenuhi.
Jadi kesimpulannya pada fungsi divide(a, b), penggunaan asersi cocok jika fungsi tersebut hanya digunakan secara internal dengan input yang sudah terkontrol. Namun, jika fungsi tersebut menerima input dari luar, seperti dari user, maka penggunaan eksepsi lebih disarankan agar sistem tetap stabil dan tidak mengalami crash secara tiba-tiba.
